DESIGN SYSTEM
Rahiq · FiqhNafs

Système de design commun aux plateformes رحيق المعرفة et FiqhNafs. Palette chaude, typographie mixte arabe/latine, dark mode natif, composants RTL-first.

1. Principes de design

Les décisions qui gouvernent chaque choix visuel.

🌡️
Chaleur & neutralité
Fond crème (#F6F3EE) plutôt que blanc pur. Les surfaces warm réduisent la fatigue oculaire lors de longues lectures.
🌙
Dark mode natif
Chaque token a sa valeur dark. Jamais de simple inversion : le dark mode utilise des teintes chaudes nuit (#1C1B19).
📖
Lecture first
Taille de base 15–17px, line-height 1.8–2.1 en arabe. Le corps de texte arabe utilise Noto Naskh pour une lisibilité maximale.
↔️
RTL first
Toute la structure est dir="rtl". Le latin (chiffres, labels UI) utilise la police sans-serif latine avec .lat.
🔴
Un seul accent
Rouge terre cuite (#C8341B) comme couleur d'action principale. Navy (#24386B) en secondaire. Éviter la surcharge de couleurs sémantiques.
⚡
Transitions courtes
Toutes les animations ≤ 250ms. Fade + translateY(4px) pour les écrans, .15s ease pour les états hover.

7. RTL & bilingue

La plateforme est dir="rtl" nativement. Règles à respecter.

RègleImplémentation
Document RTL <html lang="ar" dir="rtl"> — toute la structure s'inverse automatiquement
Texte latin dans RTL Classe .lat → font-family: --f-sans. Ne pas forcer dir="ltr" sauf champs de saisie spéciaux.
Chiffres / stats Classe .mono → --f-mono + font-feature-settings:"tnum" + letter-spacing:.01em
Icônes directionnelles Flèches (→ ←) : utiliser les caractères Unicode plutôt que SVG pour l'auto-mirroring RTL
Bordures directionnelles Utiliser border-right (côté logique start en RTL) pour les accents de paragraphe
margin-inline-start Préférer les propriétés logiques CSS (margin-inline-start, padding-inline-end) aux marges physiques
Champs de saisie text-align: right pour les inputs arabes. Les inputs latins (clé de sync) gardent l'alignement gauche.
